GITHUB: INFORMACIÓN Y CONCEPTOS

La plataforma Github y el software Git tienen diferencias importantes las cuales hay que aprender a diferenciar con el fin de implementarlas en conjunto en el desarrollo de sitios web,esto gracias a que la plataforma nos permitirá crear repositorios con los cuales podremos fomentar el trabajo colaborativo, la elección de roles y nos mantendrá el repositorio actualizado viendo el trabajo que cada una de las partes esta realizando, por su lado el software de Git nos permitirá tener un control de las versiones y asi podremos ver como mejora nuestro sitio, en esta sección es específico se hablará de la plataforma de Github.


¿Qué es GitHub?

Facilita la colaboración con git. Es una plataforma que puede mantener repositorios de código en almacenamiento basado en la nube para que varios desarrolladores puedan trabajar en un solo proyecto y ver las ediciones de cada uno en tiempo real.

¿Cuál es el origen de Git?

"Nacido en 2008 en una oficina del Valle de San Francisco (como tantos otros proyectos), GitHub tiene hoy más de 9 millones de usuarios registrados, más de 200 millones de visitas al mes y ha sido valorado en más de 2000 millones de dólares (por lo que está en la Unicorn List de Fortune). Utilizan GitHub para gestionar y almacenar su código desde Google a la Casa Blanca pasando por Facebook o incluso el Ayuntamiento de Madrid. En la capital finesa Linus Torvalds, el creador de Linux, se enfrentaba a un problema de sobredimensionamiento en su universal proyecto: demasiados desarrolladores, demasiadas funcionalidades." Github tiene dos formas de acceso, una es desde el escritorio de nuestro ordenador Gthub Desktop y otro es desde el sitio web "Github Web", el Github en el escritorio nos permitirá actualizar los repositorios creados con Git, allí con cada modificación que hagamos en el código de las distintas páginas o archivos que conforman el repositorio debemos comentarlo con la sintesis de lo que estamos actualizando o lo que se hizo, al subirlo del repositorio local al repositorio remoto "push" ya podremos acceder a el desde la web y asi mismo es necesario recordar que al revisar la configuración del repositorio en la red podremos habilitar el link de Github Pages, para acceder a el como una verdadera página web. Es necesario recordar que podemos denominar a Github como la red social de los desarrolladores, a continuación puntos claves de sus beneficios:


  • "Fomenta el trabajo colaborativo"
  • "Asignación de roles"
  • "Observar aportes de los colaboradores"
  • "Cuenta con la funcionalidad "fork" que permite clonar repositorios ajenos y si hacemos modificaciones en el podemos usar la función "pull" para que el dueño del proyecto inicial pueda analizar las modificaciones e incluirlas"
  • "No solo es para desarrolladores, sino también para aquellos que tengan un proyecto en mente y que necesitan mantener actualizado"
